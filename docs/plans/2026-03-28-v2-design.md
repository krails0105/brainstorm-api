# v2.0 구현 설계 문서

## 개요

v2.0 마일스톤 (3/30 ~ 4/12). 룸 내 아이디어 활동 — 채팅, 아이디어, 코멘트, 투표.

## 1. 채팅 (WebSocket)

### ChatMessage 엔티티

| 필드 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| room | Room (ManyToOne) | 어떤 룸의 메시지 |
| user | User (ManyToOne) | 누가 보냈는지 |
| content | String | 메시지 내용 |
| createdAt | LocalDateTime | 전송 시간 |

### 기술

- Spring WebSocket + STOMP 프로토콜
- 구독: `/topic/rooms/{roomId}` — 룸별 메시지 브로드캐스트
- 발행: `/app/rooms/{roomId}/chat` — 메시지 전송
- DB 저장: 메시지 수신 시 ChatMessage 저장 (v3.0 AI 도출용)
- 이전 대화 조회: `GET /api/rooms/{roomId}/chat` — REST API로 히스토리 조회

### 새로 생성할 클래스

| 클래스 | 역할 |
|--------|------|
| WebSocketConfig | WebSocket + STOMP 설정 |
| ChatController | WebSocket 메시지 핸들러 |
| ChatRestController | 이전 대화 조회 REST API |
| ChatService | 메시지 저장/조회 |
| ChatMessage (엔티티) | 채팅 메시지 |
| ChatMessageRepository | DB 접근 |

## 2. 아이디어

### Idea 엔티티

| 필드 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| room | Room (ManyToOne) | 어떤 룸의 아이디어 |
| user | User (ManyToOne) | 누가 만들었는지 |
| keyword | Keyword (ManyToOne, nullable) | 키워드에서 파생된 경우. 독립 생성이면 null |
| content | String | 아이디어 문장 |
| createdAt | LocalDateTime | 생성일 |

### API

| 메서드 | 경로 | 설명 | 권한 |
|--------|------|------|------|
| GET | `/api/rooms/{roomId}/ideas` | 아이디어 목록 조회 (좋아요 정보 포함) | 인증된 유저 |
| POST | `/api/rooms/{roomId}/ideas` | 아이디어 생성 | 인증된 유저 |
| PUT | `/api/rooms/{roomId}/ideas/{ideaId}` | 아이디어 수정 | 본인 or owner |
| DELETE | `/api/rooms/{roomId}/ideas/{ideaId}` | 아이디어 삭제 | 본인 or owner |

### IdeaRequest DTO

```json
{
  "content": "AI 기반 실시간 번역 서비스",
  "keywordId": 1  // nullable, 키워드에서 파생 시
}
```

### IdeaResponse DTO

```json
{
  "id": 1,
  "content": "AI 기반 실시간 번역 서비스",
  "nickname": "홍길동",
  "keywordId": 1,
  "likeCount": 3,
  "liked": true
}
```

| 필드 | 타입 | 설명 |
|------|------|------|

### API

|--------|------|------|
| POST | `/api/ideas/{ideaId}/like` | 토글 (없으면 추가, 있으면 삭제) |
